Combat system

Token / status system

Designed and implemented a token/status system that drives combat flow and interacts with turn order, stats, and abilities.

  • Tokens influence turn order calculations and stat evaluation during combat
  • Applied to both player and enemy flow for predictable but flexible interactions
  • Supports combat readability via clear state changes and consistent rules

AI logic and targeting

Implemented boss logic and targeting behavior to make encounters feel distinct and reactive.

  • Boss AI logic tied into combat state (tokens, turn order, current threats)
  • Targeting rules that respond to party state and status effects
  • Combat decisions designed to be understandable and tuneable

Save/load and persistence

Built save/load for the full game, designed for overworld progression and persistent state across scenes.

  • Overworld saving including player position, party state, inventory, equipment, and stats
  • Supports progression data such as quests/objectives and game state changes
  • Designed to load reliably across scene transitions and long play sessions
What this enables: stable long-term saves, consistent state after reload, and a foundation for autosaves/slots.

Equipment system

Two-slot equipment with modifiers

Implemented a character equipment system with two slots and item-driven modifiers (positive and negative).

  • Items modify stats directly through clear modifiers
  • Supports tradeoffs (power vs drawbacks) to shape builds and party strategy
  • Designed to be extendable as item variety grows

Keyword-driven interactions

Some equipment affects combat rules via keywords, including interactions with tokens/status effects.

  • Example: bonus damage or altered behavior against specific states (e.g., stunned)
  • Rules are consistent and easy to tune during balance passes
  • Creates build identity beyond raw stat increases

Map systems

World map progression

Built a world map system that unlocks as the player progresses through the game.

  • Progress-based unlocking to support exploration pacing
  • UI integration designed for clarity and quick navigation

Minimap + objectives

Implemented minimap functionality to support navigation and highlight quest objectives.

  • Objective indicators to reduce friction and improve player guidance
  • Designed to be easy to extend for new quest/objective types

Tools and tech

  • Engine: Unity
  • Language: C#
  • Data: JSON-based save/load & persistence
  • Focus areas: gameplay systems, combat, AI, UI integration, world/map systems
